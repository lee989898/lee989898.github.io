DATA MANIPULATION LANGUAGE (DML)

DATA MANIPULATION LANGUAGE (DML)

  • Data Manipulation Language (DML)
    • Database의 data를 조작(schema는불변)하는언어.
    • Retrieve
    • Insert
    • Drop Table ex) drop table r ex) insert into instructor values (‘10211’, ‘Smith’, ‘Biology’, 66000);
    • Delete Remove all tuples from the student relation ex) delete from student
    • Change
    • Alter

주의: SQL 이름은 대소문자 구분하지 않는다.

SELECT

데이터베이스에서 원하는 데이터를 검색, 추출할때 사용한다.

SQL은 쿼리 결과뿐만 아니라 관계에서도 중복을 허용한다.

중복을 제거 하고 싶으면 distint 키워드를 사용하면 된다.

select distinct dept_name
from instructor

모든 결과를 보고 싶으면 all 키워드를 사용하면 된다.

select all dept_name
from instructor

*표시는 모든 속성을 나타낸다.

select *
from instructor

속성은 from 절 유무에 상관없이 리터럴을 사용할 수 있다.

select '437'

select  ‘A’
from instructor

select절은 수학 표현(+,-,*,/)를 포함할 수 있다.

selectID, name, salary/12
from instructor

SELECT ename, (sal+200) * 12 FROM emp;
SELECT ename, -sal * 10 FROM emp;

salary/12는 salary/12 as monthly_salary 처럼 rename할 수 있다.

WHERE

조건을 부여하여 만족하는 ROW Selection

연산자

  • =, !=, >, <, <=, >=
  • IN : 집합에포함되는가?
  • BETWEEN a AND b : a 와b 사이?
  • LIKE: 문자열부분검색
  • IS NULL, IS NOT NULL: NULL인지 검색
  • AND, OR: 둘다 만족? 둘 중 하나만 만족?
  • NOT: 만족하지않음?
  • ANY, ALL : 집합중어느한열, 집합중모든열(다른비교연산자와함께사용)
  • EXIST: 결과Row가하나라도있나? (subquery에서)

To find all instructors in Comp. Sci. dept

select name
from instructor
where dept_name='Comp. Sci.'

To find all instructors in Comp. Sci. deptwith salary > 80000

select name
from instructor
where dept_name='Comp. Sci.' and salary > 80000

FROM

from절은 쿼리와 관련된 관계를 나열한다. 관계 대수의 데카르트 곱 연산에 해당한다.

select * 
from instructor, teaches

가능한 모든 instructor와 teaches 쌍을 생성한다. 공통 속성(예: ID)의 경우 결과 테이블의 속 성은 관계 이름(예: instructor.ID)을 사용하여 이름이 바뀝니다.

카티션 프로덕트는 그냥 사용하면 유용하지 않지만 where절 조건과 결합하면 유용하다.

Find the names and the course_id of all instructors who have taught some course

select name, course_id
from instructor, teaches
where instructor.ID = teaches.ID

Find the names of all instructors in the Art department who have taught some course and the course_id

select name, course_id
from instructor, teaches
where instructor.ID = teaches.ID and
      instructor. dept_name= 'Art'

ADDITIONAL BASIC OPERATIONS

  • Rename

    old-name as new-name

Find the names of all instructors who have a hi gher salary than some instructor in ‘Comp. Sci’.

select distinct T.name
from instructor as T, instructor as S
where T.salary> S.salary and S.dept_name= 'Comp. Sci.'

as는 선택적이다. instructor as T ≡ instructor T

칼럼 제목 변경 큰따옴표(“ “)을 사용하여 alias내에 공백이나 특수문자를 포함할수있다.

SELECT enamename FROM emp;
SELECT enameas name FROM emp;
SELECT ename“as” FROM emp;
SELECT (sal+ comm) “Annual Salary” FROM emp;
  • LIKE

Wildcard를 이용한 문자열 부분 매칭

Wildcard % : 임의의 길이의 문자열(공백 문자 가능) _ : 한 글자

Escape: ESCAPE 뒤의 문자열로 시작하는 문자는 Wildcard가 아닌것으로 해석

ex)

  • ename LIKE ‘KOR%’ : ‘KOR’로 시작하는 모든 문자열(KOR가능)
  • ename LIKE ‘KOR_’ : ‘KOR’ 다음에 하나의 문자가 오는 모든 문자열
  • ename LIKE ‘KOR\%%’ ESCAPE ‘' : ‘KOR%’로 시작하는 모든 문자열

  • Literal

  • SELECT 절에 사용되는 문자, 숫자, Date 타입 등의 상수
  • Date 타입이나 문자열은 작은따옴표(‘ ‘)로 둘러싸야 함
  • 문자열 결합(Concatenation) 연산자:  

ex):

  • SELECT ‘Name is ‘   ename   ’ and no is ‘   empno FROM emp;
  • ORDER BY

주어진 컬럼 리스트의 순서로 결과를 정렬

결과 정렬 방법

  • ASC: 오름차순(작은값→큰값) (default)
  • DESC: 내림차순(큰값→작은값)

컬럼이름대신 Alias, expr,SELECT 절상에서의 순서 (1, 2, 3…)도사용가능 ex) 부서번호 순으로 정렬하고, sal가 높은 사람부터 출력하시오 SELECT * FROM emp ORDER BY deptno, sal DESC

  • BETWEEN

ex) Find the names of all instructors with sala ry between $90,000 and $100,000 (that is , >= $90,000 and <= $100,000)

select name
from instructor
where salary between 90000 and 100000
  • Data Control Language (DCL)
    • Database의 constraint를 제어하는 언어.
  • Procedural vs nonprocedural languages
    • 절차적
      • 필요한 데이터와 해당 데이터를 얻는 방법 지정
      • e.g., Relational Algebra
    • 비절차적
      • 선언적
      • specify only what data are needed
      • e.g., Relational Calculus, SQL
    • Query (≈ Question)
      • 정보 검색 요청
      • query language: part of DML
      • sometimes “query language = DML”

  • Consistency Constraints(일관성 제약)

  • Domain Constraints(도메인 제약 조건)
    • 가능한 값의 도메인은 모든 속성이다.
    • E.g., integer types, character types, etc
  • Referential Integrity(참조 무결성)
    • 한 관계에 나타나는 값은 다른 관계에도 나타난다.
  • Assertions
    • 데이터베이스가 항상 충족해야 하는 모든 조건
    • E.g., “모든 학과는 매 학기에 최소 5과목을 이수해야 합니다.”
  • Authorization(권한 부여)
    • 사용자마다 다른 권한
  • SQL

SQL은 널리 사용되는 비절차적 언어이다. SQL은 튜링 머신과 동등한 언어가 아니다. 복잡한 기능을 계산할 수 있도록 SQL은 일반적으로 일부 고급 언어에 내장되어 있습니다.

응용 프로그램은 일반적으로 SQL 쿼리를 데이터 베이스로 보낼 수 있는 응용 프로그램 인터페이스 (예: ODBC/JDBC)를 허용하기 위해 언어 확장 중 하나를 통해 데이터베이스에 액세스합니다. ( Java Database Connectivity) 표준은 Java 언어에 해당하는 기능을 제공합니다.

  • 트랜잭션 데이터베이스 응용 프로그램에서 단일 논리 기능을 수행하는 작업 모음이다. 프로그래머는 “정확한” 트랜잭션 작성에 대한 책임이 있다.

DBMS는 BMS는 각 트랜잭션의 ACID를 보장해야 합니다

  • atomicity: 전부 아니면 전무
  • consistency: 결과가 정확해야 함
  • isoloation: 변경 사항이 만들어지고 다른 사람이 볼 수 있는 방법과 시기를 제어합니다.
  • durability: 트랜잭션 성공 후에도 효과가 지속되어야 합니다.
