ViewBinding

DataBinding을 사용하면 코드의 데이터를 뷰에 바인딩하고 뷰를 코드에 바인딩할 수 있다. 그러나 프로젝트의 요구 사항이 뷰를 코드에 바인딩하는 것이라면 annotation 프로세서는 빌드 시간에 영향을 미치기 때문에 데이터 바인딩이 원하는 만큼 효율적이지 않다

그래서 그 문제에 대한 해결책으로 ViewBinding이 “What’s new in Android talk”에서 소개됐다. 2019년 Google IO에서 데이터 바인딩이 할 수 없는 일을 뷰 바인딩이 할 수 있는 것은 없다. 뷰 바인딩을 사용하여 코드의 데이터를 뷰에 바인딩할 수 없다

그러나 데이터 바인딩에 비해 뷰 바인딩에는 두 가지 장점이 있다. 뷰 바인딩은 사용하기가 매우 쉽다. 뷰 바인딩은 XML에 레이아웃 태그를 특별히 추가할 필요가 없다. 또한 xml 뷰에 표현식을 추가할 필요가 없다. 실제로 xml 파일을 특별히 편집할 필요는 전혀 없다. 뷰 바인딩에는 annotation이 필요하지 않기 때문에 컴파일 시간이 더 빠르다

뷰 바인딩을 사용하려면 우선 앱 수준 build.gradle 파일에서 뷰 바인딩을 활성화해야 한다. 더 큰 프로젝트에 대해 데이터 바인딩과 뷰 바인딩을 모두 활성화된 상태로 유지하는 것이 좋다

코드 
// build.gradle
buildFeatures {
    viewBinding true
}

프로젝트의 프로젝트 요구 사항을 적절하게 분석한 후, 데이터 바인딩에서 제공하는 고급 기능이 필요하지 않다는 것을 인식하면 뷰 바인딩을 사용하면 좋다

서비스

Android에서 서비스는 백그라운드에서 장기 실행 작업을 수행할 수 있는 애플리케이션 구성 요소이다. 서비스는 사용자 인터페이스를 제공하지 않아서 액티비티 또는 프래그먼트 수명 주기에 연관되지 않는다. 응용 프로그램이 닫히거나 사용자가 전환하더라도 서비스는 백그라운드에서 계속 실행될 수 있다

간단한 앱을 사용하여 서비스의 기본 사항을 알아보자

먼저 뷰 바인딩을 사용하기 위해 추가하자

코드 
// build.gradle
buildFeatures{
        viewBinding true
    }

다음으로 서비스 클래스를 생성한다. 이 클래스는 상위 서비스 클래스를 확장해야 한다. 다음으로 onBind 기능을 구현해야 합니다. 서비스를 생성할 때마다 onBind 기능을 구현해야 한다. 그러나 지금은 사용하지 말자. 이제 init 블록을 만들고 서비스가 생성되고 있음을 나타내는 로그 메시지를 거기에 추가한다

그런 다음 해당 서비스를 시작하려면 메인액티비티 버튼에서 startService() 함수를 호출해야 한다 여기에서 클라이언트가 무언가를 요청할 때마다 필요한 작업을 수행하기 위해 코드를 작성해야 한다
여기에서 기본 반환 값을 변경해야 한다. 많은 옵션이 있음을 알 수 있다. 이 반환 값은 Android 시스템에서 시스템이 서비스를 다시 시작하는 방법을 결정하는 데 사용된다

START_REDELIVER_INTENT을 선택하면 시스템은 다시 시작할 때 마지막 의도를 서비스에 전달한다
START STICKY를 선택하면 시스템이 다시 시작할 때 마지막 의도를 전달하지 않는다 START NOT STICKY 시스템은 서비스를 다시 시작하지 않는다

다음으로, 우리는 onDestroy 기능을 구현 한다. 메인액티비티에서 stopService() 함수를 호출 할 때 호출 된다

컴패니언 객체는 클래스에 연결되지만 해당 클래스의 인스턴스에는 연결되지 않는다

코드 
class MyBackgroundService: Service() {

    init {
        Log.d(TAG, "Service has been created")
    }

    override fun onStartCommand(intent: Intent?, flags: Int, startId: Int): Int {
        Log.d(TAG, "Service Started")
        val name = intent?.getStringExtra(NAME)
        val marks = intent?.getIntExtra(MARKS, 0)
        Log.i(TAG, "name is $name , marks : $marks")

        return START_STICKY
    }

    override fun onBind(intent: Intent?): IBinder? = null

    override fun onDestroy() {
        Log.d(TAG, "Destroying..")
        super.onDestroy()
    }

    companion object {
        const val TAG = "BACKGROUND"
        const val NAME = "NAME"
        const val MARKS = "TOTAL MARKS"
    }
}

다음으로 뷰에서 서비스로 추가 데이터를 보내고 서비스에서 데이터를 받자

코드 
class MainActivity : AppCompatActivity() {

    private lateinit var binding: ActivityMainBinding
    override fun onCreate(savedInstanceState: Bundle?) {
        super.onCreate(savedInstanceState)
        binding = ActivityMainBinding.inflate(layoutInflater)
        setContentView(binding.root)

        val serviceIntent = Intent(this, MyBackgroundService::class.java)
        serviceIntent.putExtra(MyBackgroundService.NAME, "Alex")
        serviceIntent.putExtra(MyBackgroundService.MARKS, 78)

        binding.btnStart.setOnClickListener {
            Log.i(MyBackgroundService.TAG, "Starting Service")
            startService(serviceIntent)
        }

        binding.btnStop.setOnClickListener {
            Log.i(MyBackgroundService.TAG, "Stopping Service")
            stopService(serviceIntent)
        }

    }
}

class MyBackgroundService: Service() {

    init {
        Log.d(TAG, "Service has been created")
    }

    override fun onStartCommand(intent: Intent?, flags: Int, startId: Int): Int {
        Log.d(TAG, "Service Started")
        val name = intent?.getStringExtra(NAME)
        val marks = intent?.getIntExtra(MARKS, 0)
        Log.i(TAG, "name is $name , marks : $marks")

        return START_STICKY
    }

    override fun onBind(intent: Intent?): IBinder? = null

    override fun onDestroy() {
        Log.d(TAG, "Destroying..")
        super.onDestroy()
    }

    companion object {
        const val TAG = "BACKGROUND"
        const val NAME = "NAME"

    }
}
