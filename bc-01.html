블록체인이란

블록체인은 왜 어려운가?

Cryptography

  • Public Cryptography(공개키 암호학)
  • Secure Hashing

암호기술(AES, DES, …)은 원칙적으로는 블록체인과 상관이 없다

Security

  • 보안의 3대요소
    • C - Confidentiality(인가된 당사자에 의해서만 접근하는 것을 보장, 기밀성)
    • I - Integrity(인가된 당사자에 의해서 인가된 방법로만 변경가능한 것, 무결성)
    • A - Availability(적절한 시간에 인가된 당사자에게 접근 가능해야 하는 것, 가용성)
  • Authenticity(인증)
  • Non-repudiation(부인을 할 수 없다)

Distributed Consensus(분산 합의, Byzantine)

  • Proof-of-Work
  • Proof-of-Stake
  • Probaility(확률론)

P2P Networking

Programming languages

  • Solidity
  • Vyper
  • Go
  • Javascript

Complex operation mechanisms

Peer-to-Peer Networking (P2P)

Communication Models

  • Point-to-Point
  • Client-Server(HTTP(S), Centralized architecture, 중압집중적)
  • Peer-to-Peer(Decentralized architecture, 비집중형, Client가 될 수도 Server가 될 수도)
    • Scalability
    • No Single-point-of-failure
    • No governance
    • Distributed Architecture과는 다르다

Operations of P2P Network

네트워크 구성

  • 주변/랜덤 노드들과의 연결을 통한 그래프 형성

정보 전달

  • Flooding: 정보를 전체 네트워크에 Broadcast
  • Unicasting: 특정 노드로 메시지 전송 (Routing protocols)

네트워크 참여

  • 누구나 참여가능
  • 오픈 네트워크(public)
  • Join/Leave protocol

Blockcahin protocols

  • Bitcoin Network
  • 이더리움: ÐΞVp2p protocol

Security Principles & Blockchain

  • 보안의 3대요소
    • C - Confidentiality (보장하지 않음)
    • I - Integrity ([BC] Transaction, Block, Address)
    • A - Availability (Denial of Service (Dos) 공격)
  • Authenticity (Signature 로 Ownership 증명 (Anonymous))
  • Anonymity (익명성, One time public key)
  • Non-repudiation (Signature)

Public Key Cryptography (공개키 암호)

대칭키 암호(Symmetric)(DES, 3DES, AES, ARIA)

비대칭키 암호(Asymmatric)(RSA, ECC)

발상의 전환 1: 누구나 나에게 암호문을 보낼 수 있다(키배분)

발상의 전환 2: 누구나 내 암호문을 풀 수 있다?(전자서명)

공개키쌍(Key pair)

  • Public Key(공개키): 누구나 알 수 있음, 암호에 사용, 서명 확인에 사용

  • Private Key(개인키): 소유자만 알고 있음, 복호화에 사용, 서명에 사용

  • What is Public key Infrastructure?

공개키암호의 한계: 주어진 공개키가 누구의 것인가가 불분명 인증서: 신뢰자가 공개키의 주인을 증명하는 문서(발급자 서명) 발급자의 공개키는 누가 인증할것인가? 풀 수 없는 문제

  • 상대방의 공개키를 확실하게 확인하는 방법 PKI

Public Key Crypto in Blockchain

  • 서명알고리즘 사용
  • Asset에 대한 Ownership을 증명하는데 사용
    • A로부터 받은 돈을 B가 사용할 떄 B가 서명으로 소유권 증명
    • B가 생성/설치한 Smart Contract에 대한 생성자 증명
  • Proof-of-Authority (PoA)
    • Ethereum의 변형으로 일부 testnet에 사용 (Kovan)
    • 서명확인된 Miner만 블록생성 가능
  • 서명 알고리즘
    • ECC (Elliptic Curve Cryptography) 기반 서명

Elliptic Curve

  • Elliptic Curve
    • $y^2$ = $x^3$ + ax + b
    • 곡선의 두 점을 이으는 선을 그리면 반드시 한점에서만 곡선과 교차
  • Elliptic Curve Algebra (대수)
    • P1 + P2 = P3
    • 0(항등원) = Infinity point
  • Elliptic Curve Cryptography
    • 큰 솟수(p)의 필드(Fp) 기반
  • secp256k1 -> bitcoin
    • $y^2$ = ($x^3$ + 7) over ($F_p$)
    • p = $2^256$ - $2^32$ - $2^9$ - $2^8$ - $2^7$ - $2^6$ -$2^4$ - 1

Elliptic Curve Cryptography (ECC)

  • Given
    • Elliptic Curve (a, b, p)
    • Generator G
  • Pick a random number K (엄청 큰 수)
  • Comput K*G
  • Public key(x,y) = K * G
  • Private key = K
  • KG 로부터 K를 구할 수 없음
  • RSA보다 10배 적은 키로 비슷한 보안성 유지

Hash & Cryptographic Hash

  • Hash
    • 임의의 길이의 Input을 고정된 길이 output으로 변환
    • Input값들을 균등하게 변환시켜줌
    • Constant-time search를 위해서 사용
    • EX: mod p
  • Cryptographic Hash (Secure Hash)
    • One-way function
    • Each to compute
    • Collision Resistance (weak or strong)
      • 같은 Hash값을 갖는 input을 찾을 수 없다
  • Bitcoin uses SHA256
  • Ehtereum sues Keccak256(SHA3 원형)

Proof of Work puzzle

  • 주어진 d에 대해서, 그 Hash값이 d개 이상의 0으로 시작하도록(<$2^{256-d}$)하는 M을 찾는 문제
  • Secure hash에 대한 유일한 해답은 brute-force 방법
  • d(difficulty)가 증가할 수록 기하급수적으로 오래걸림

Origin of Proof-of-Work: Hashcash

  • Cynthia Dwork에서 제안함
  • Hashcash는 Adam Back이 제안함
  • 이메일을 보내면 PoW 비용이 발생한다
    • < counter >를 찾는다. SHA-1(1:20:< time >:< recipient >:< rand >:< counter >)=$0^{20}${0,$1^{140}$}
    • 1초 정도 걸린다
  • 리시버는 이메일을 받는다 오직
    • < time >이 최근(2-days), < recipient >가 맞을떄
    • 그리고 유효한 PoW일때
  • 이것으로 스팸을 예방할 수 있다

Merkle Tree

  • 다수의 데이터(tx)의 hash를 계산하되, 특정 데이터(tx)의 포함 여부를 빠르게 (log N) 검사할 수 있는 기술
  • Block Header에 transaction Merkle 해시값만 저장
  • HK의 포함확인을 위해서 모든 15개 다른 데이터가 필요하지 않고 4개의 데이터만 필요

Cryptocurrency

Electronic Money Problem

  • Electronic money
    • 모든 Cash Flow를 추적하여 모든사람의 자산을 파악 (Transaction)
    • 모두가 신뢰할 수 있는 정보 (Consensus)
  • Easy solution
    • 모든 Transaction을 Trusted Third Party(TTP)를 통해서 실행
    • TTP에 대한 신뢰가 필요
  • Hard solution
    • distributed solution (누구의 도움도 없이 거래. 현재 Cash와 같음)
  • 왜 어려울까?
    • Double spending(중복지불)을 감지할 수 없음
    • Bitcoin은 consensus algorithm으로 해결

How Bitcoin works (without Why)

  • cash flow를 transaction chain에 기록
  • 유효한 tx(no double spending)은 block에 저장
  • 각 block은 chain으로 연결됨
  • 모두(대다수)가 합의한 (가장 긴)block chain만 신뢰
  • Hash-point ->
    • 이전 tx/block의 hash 값을 다음 tx/block에 기록
    • 중간의 tx/block이 수정되면 연결이 끊어짐
  • P2P 네트워크로 tx/block 교환
  • 각 tx은 지불자가 서명
  • 각 tx은 수신자의 공개키(의 hash) 표기
  • Miner가 유효한 tx를 모아서 block 생성
  • 가장 긴 blockchain에 추가 (append)
  • PoW을 푼 block만 유효함
  • Longest block-chain이 consensus 획득

Why Bitcoin works

  • 공개 암호화 (ECCDSA)
    • Authenticity, Non-repudiation
  • Cryptographic hash (SHA256/ RIPE256)
    • 트랜잭션 및 블록의 무결성
  • 합의 알고리즘 (Prrof of Work)
    • 민주적 진실성
    • 공격자는 자신의 취향에 맞는 블록체인을 만들 수 없다
      • 공격자가 다른 사람보다 수적으로 열등하다

Beyond Cryptocurrency

After Bitcoin

  • MatkeyCap
    • 단위가치 * 총 volume
    • Bitcoin > Ethereum > Ripple
  • 대부분이 Bitcoin, Litecoin, Ethereum의 fork버전
  • 차이점
    • Scripting language
    • Transaction 관리
    • 추가기능 및 보안기능
    • Mining 여부
    • Consensus 알고리즘
    • 단순 파라미터 차이

Taxonomy of Altcoins

  • Purposes
    • Cryptocurrency-only: Bitcoin
    • Cryptocurrency & Business logic: Ethereum
    • Business logic only: Hyperledger
  • Accessibility
    • Public block chain
    • Private block chain
    • Permissioned (consortium) block chain

Namecoin

  • 분산 도메인 이름 서비스
  • 첫번째 Altcoin(2011)
  • “*.bit” 웹사이트용 Chrome/Firefox용 플러그인
  • 싸다 (5 cents) for, no cost
  • 도메인 판매 가능
  • 병합 채굴

Litecoin

  • 첫 번째 메모리 하드 마이닝 퍼즐 (2011)
    • Bitcoin: 계산이 어려운 마이닝 퍼즐
    • 블록이 2.5분마다 생성 (Bitcoin: 10min)
    • 두번째로 많이 fork되었다

PeerCoin

  • 최초의 지분 증명(PoS) 채굴 퍼즐(2012)
  • Mining: 코인을 이용한 거래
    • 사용하지 않은 코인이 많을수록 당첨 확률이 높아진다
    • 실제로 PoW와 하이브리드
  • 완전히 분산되지 않음
    • 신뢰할 수 있는 주소는 체크포인트에 서명한다

DogeCoin

  • 단순히 암호화폐를 즐기기 위해(2013)
  • 많은 마케팅 캠페인
  • 랜덤 블록 보상
    • 각 블록 보너스는 무작위입니다: PRG(이전 블록 해시)
    • 예측 가능하므로 채굴자가 낮을 때 discurage 된다
    • Removed after a few months

Monero

  • 더 높은 개인 정보 보호 수준
  • 비트코인은 가명만 지원
  • Monero는 “반지 서명”을 사용
    • 여러 참가자의 조합
  • 대체 동전
    • 비트코인 추적/식별 가능
    • 추적할 수 없는 모네로 코인

Cardano

  • 최초의 입증 가능한 보안 PoS 알고리즘: Ouroboros(2015)
  • 사용자 개인 정보 보호
  • 현재 시가총액 상위 5위
  • 빠른 속도
  • Layered
    • 정산 계층: 사용자가 교환
    • 제어 계층: 신뢰할 수 있는 계산 프레임워크

Etherum

  • 최초의 스마트 계약 암호화폐(2013)
  • Turing-complete 언어로 된 스마트 계약
  • 여러 스마트 계약이 컴퓨터 프로그램을 작동
    • 네트워크에 설치됨
    • Ether로 구동되는 실행
  • 스마트 계약을 사용하는 DApp 허용
