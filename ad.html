Navigation Architecture Component

Activity와 Fragment, 화면 간 전환을 제공하는 것은 복잡한 작업이 었다
최근에 소개된 Android jetpack 탐색 아키텍처 구성 요소 라이브러리는 인앱 탐색에 필요한 모든 것을 처리하기 위해 많은 빌드 도구를 제공함으로써 훨씬 더 개발 작업을 쉽게 만들어준다

Android 개발의 새로운 추세는 단일 Activity, 다중 Fragment 모델이다 네비게이션을 사용할 때 아키텍처 구성 요소에서 일반적으로 기반 역할을 하는 하나의 빈 Activity를 만든다
그 외 모든 것들 앱의 화면은 Fragment를 사용하여 생성된다

라이브러리는 명령을 고려하여 뒤에서 많은 지원 클래스를 생성한다. 이렇게 생성된 클래스는 작업을 더 간단하고 쉽게 만든다 탐색 구성 요소에는 세 가지 주요 부분이 있다. 네비게이션 그래프, NavHostFragment 및 NavController

네비게이션 그래프는 모든 탐색 관련 정보를 포함하는 XML 리소스 파일이다/ 이를 통해 단일 위치에서 탐색 관련 작업을 관리할 수 있다 NavHostFragment는 네비게이션 그래프를 유지하는 활동을 한다 NavController는 네비게이션에 의해 생성된 클래스이다. 네비게이션 그래프에 추가한 목적지 간의 탐색을 관리하기 위한 라이브러리이다

네비게이션 아키텍처 구성 요소를 사용할 떄 프래그먼트 간에 데이터를 전달하려면 안전한 인수를 사용해야 한다 그리고 safe args 플러그인도 추가해야 한다

// buil.gradle
 def nav_version = "2.4.1"
classpath "androidx.navigation:navigation-safe-args-gradle-plugin:$nav_version"

네비게이션 그래프에서 앱의 모든 화면과 사용자가 화면에 접근할 수 있는 방법을 보여준다
목적지라고 불리운다

android stuido에서 네비게이션 그래프를 만든다. New Resource File에서 파일 이름을 nav_graph로 한다. 그 다음 Resource 유형을 네비게이션으로 선택한다

// nav_graph.xml
<?xml version="1.0" encoding="utf-8"?>
<navigation xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:app="http://schemas.android.com/apk/res-auto"
    android:id="@+id/nav_graph">

</navigation>

아직은 목적지와 NavHostFragment가 없다
nav_graph.xml design 부분에서 add a destination을 클릭하여 홈프래그먼트를 Blank로 추가해준다

// homeFragment.xml
<?xml version="1.0" encoding="utf-8"?>
<FrameLayout xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:tools="http://schemas.android.com/tools"
    android:layout_width="match_parent"
    android:layout_height="match_parent"
    tools:context=".HomeFragment">

    <!-- TODO: Update blank fragment layout -->
    <TextView
        android:layout_width="match_parent"
        android:layout_height="match_parent"
        android:text="@string/hello_blank_fragment" />

</FrameLayout>

TextView를 제거해주고 FrameLayout을 ConstraintLayout으로 교체해준다
이제 앱을 시작하면 홈프래그먼트가 실행된다
데이터 바인딩을 사용해야 하므로 xml을 layout으로 감싸준다. 그리고 데이터 바인딩 코드도 추가한다
프래그먼트에서 데이터 바인딩을 사용할 때는 루트 속성을 반환해야 한다

// HomeFragment.kt
class HomeFragment : Fragment() {

    private lateinit var binding: FragmentHomeBinding

    override fun onCreateView(
        inflater: LayoutInflater, container: ViewGroup?,
        savedInstanceState: Bundle?
    ): View? {
        binding = DataBindingUtil.inflate(inflater, R.layout.fragment_home, container, false)
        return binding.root
    }
}

action을 추가하면 fargment에서 fragment로 이동할 수 있다
action은 nav.graph.xml에서 연결해주면 id도 자동으로 만들어준다

// nav_graph.xml
<?xml version="1.0" encoding="utf-8"?>
<navigation xmlns:android="http://schemas.android.com/apk/res/android"
    xmlns:app="http://schemas.android.com/apk/res-auto"
    xmlns:tools="http://schemas.android.com/tools"
    android:id="@+id/nav_graph"
    app:startDestination="@id/homeFragment">

    <fragment
        android:id="@+id/homeFragment"
        android:name="com.lee989898.navdemo1.HomeFragment"
        android:label="fragment_home"
        tools:layout="@layout/fragment_home" >
        <action
            android:id="@+id/action_homeFragment_to_secondFragment"
            app:destination="@id/secondFragment" />
    </fragment>
    <fragment
        android:id="@+id/secondFragment"
        android:name="com.lee989898.navdemo1.SecondFragment"
        android:label="fragment_second"
        tools:layout="@layout/fragment_second" />
</navigation>

findNavController에 정해진 id를 통해서 다른 fragment로 이동한다

// HomeFragment.kt
binding.button.setOnClickListener {
            it.findNavController().navigate(R.id.action_homeFragment_to_secondFragment)
}

navController를 사용하여 지금, fragment에서 다른 fragment로 데이터를 전달할 수 있다
HomeFragment에서 SecondFragment의 TextView에 입력을 표시하는 코드를 작성한다
사용자 입력을 번들로 받는다. 이 editText의 값을 id editText로 가져온다
EditText 값의 이름을 “user_input”으로 지정한다. 그런 다음 해당 번들을 이 탐색 함수의 두 번째 인수로 전달해야 합니다 대상 간에 데이터를 전달하는 것은 권장되지 않는다. 가장 좋은 방법은 뷰 모델을 사용하는 것이다
그러나 Android Navigation 아키텍처 구성 요소를 사용하면 데이터를 전달할 수 있다

// HomeFragment
binding.button.setOnClickListener {
            val bundle: Bundle = bundleOf("user_input" to binding.editText.text.toString())
            it.findNavController().navigate(R.id.action_homeFragment_to_secondFragment, bundle)
        }

인수에서 가져오는 코드이다

// SecondFragment
val input: String? = arguments!!.getString("user_input")
    binding.textView.text = input.toString()

null 입력을 피하기 위해, EditText에 약간의 유효성 검사를 추가한다
TextUtils.isEmpty로 binding.editText.text.toString() 유효성 검사를 하여 사용자는 입력을 비워둘 수 엇ㅂ다

// HomeFragment
binding.button.setOnClickListener {
            if(!TextUtils.isEmpty(binding.editText.text.toString())){
                val bundle: Bundle = bundleOf("user_input" to binding.editText.text.toString())
                it.findNavController().navigate(R.id.action_homeFragment_to_secondFragment, bundle)
            }else{
                Toast.makeText(activity, "Please insert your name", Toast.LENGTH_LONG).show()
        }

    }

nav_graph.xml에서 action_homeFragment_to_secondFragment, action_homeFragment_to_secondFragment 이 두 속성 창에서 4개의 애니메이션을 위한 공간이 있음을 알 수 있다. (입력, 종료, 팝업 입력 및 팝업 종료)

한 목적지에서 다른 목적지로 또는 한 화면에서 다른 화면으로 이동할 때 새 화면이 나타난다
Enter 애니메이션과 함께 이전 창이 종료 애니메이션과 함께 사라진다
사용자가 전화기의 뒤로 버튼을 클릭하면 화면이 뒤로 이동하면 현재 화면이 사라진다
팝업 종료 애니메이션과 함께 이전 창이 팝업 입력 애니메이션과 함께 나타난다

지금 나의 앱은 특별한 애니메이션을 보여주지 않는다. 그저 화면이 나타나고 사라진다. 따라서 앱 화면에 애니메이션을 추가해 보겠다

안드로이드에서 애니메이션은 애니메이션 리소스 파일에 xml로 작성된다. 가장 먼저, 프로젝트에 애니메이션 리소스 파일을 추가해야 한다
검색하면 Android 애니메이션 리소스 파일용 Google, 많은 xml 샘플을 찾을 수 있다. stack over flow에서 좋은 애니메이션 리소스 파일 세트를 찾았다

push_down_in, push_down_out, push_up_in, push_up_out, slide_in_left, Slide_in_right, slide_out_left 및 slide_out_right ,rotation, scale_from_corner, scale_torwards_corner, 페이드인과 페이드아웃 등 많은 애니메이션 리소스 파일이 있다

나는 Slide_in_left, slide_in_right, slide_out_left 및 slide_out_right 기능을 추가하겠다
앱을 선택하고, 새로운 Android 리소스 파일. 웹에서 파일 이름과 xml 코드를 복사하여 붙여넣고 있다
리소스 유형은 애니메이션이어야 한다
이제 프로젝트에 애니메이션을 추가해 보겠다
나는 Slide_in_left로 애니메이션 입력을 선택했다. 그리고 종료 애니메이션을 slide_out_right로 선택했다 뒤로 버튼의 경우 팝 입력 애니메이션을 slide_in_right로 선택하고 팝 종료 애니메이션으로 Slide_out_left을 선택했다

// slide_in_left.xml
<?xml version="1.0" encoding="utf-8"?>
<set xmlns:android="http://schemas.android.com/apk/res/android"
    android:shareInterpolator="false">
    <translate
        android:duration="5000"
        android:fromXDelta="-100%"
        android:toXDelta="0%" />
    <alpha
        android:duration="5000"
        android:fromAlpha="0.0"
        android:toAlpha="1.0" />

</set>

애니메이션이 조금 느리면. 그럼 애니메이션 리소스 파일을 열어서 시간을 줄이면된다

// nav_graph.xml
<fragment
        android:id="@+id/homeFragment"
        android:name="com.lee989898.navdemo1.HomeFragment"
        android:label="fragment_home"
        tools:layout="@layout/fragment_home" >
        <action
            android:id="@+id/action_homeFragment_to_secondFragment"
            app:destination="@id/secondFragment"
            app:enterAnim="@anim/slid_in_left"
            app:exitAnim="@anim/slide_out_right"
            app:popEnterAnim="@anim/slide_in_right"
            app:popExitAnim="@anim/slide_out_left" />
    </fragment>
