__3차 세미나__

3차 세미나

ViewPager2

ViewPager는 하단 탭을 클릭하거나 화면을 스와이프하면 화면이 바뀌는 걸 말한다
화면이 넒은 데스크탑의 경우에는 한 화면에서 뷰를 전부 보여줄 수 있지만 휴대폰에서 한 화면에서 모두 보여주기에는 한계가 있어 ViewPager를 사용한다
그리고 콘텐츠마다 Activity를 만들면 원하는 Activity까지 이동하는데 오래 걸리고 앱 사이즈도 커진다
교체되는 부분 화면(Fragment) + 부분 화면을 전환하는ViewPager2 를 이용하면 한화면에서 여러개의 화면을 볼 수 있다
ViewPager2가 아닌 ViewPager도 있지만 공식문서에서 ViewPager2를 사용하는게 좋다고 한다

ViewPager

  • 수평 스크롤만 지원(가로)
  • LTR만 지원
  • 페이저 어뎁터 사용
  • 프래그먼트 교체가 어려움

ViewPager2

  • 수평, 수직 스크롤 모두 지원(가로, 세로)
  • LTR, RTL 모두 지원
  • 리사이클러뷰 기반으로 동작(리사이클러뷰 어뎁터 또한 사용 가능)
  • NotifyDataSetChanged 이용해서 UI 업데이트(프래그먼트 교체) 가능

3.Adapter 만들기

// TestViewPagerAdapter.kt
class TestViewPagerAdapter(fragmentActivity: FragmentActivity): FragmentStateAdapter(fragmentActivity) {

    val fragments = mutableListOf<Fragment>()

    override fun getItemCount(): Int = fragments.size

    override fun createFragment(position: Int): Fragment = fragments[position]

}

FragmentStateAdapter 클래스를 상속하고 이 떄 FragmentStateAdapter 클래스는 생성자로 FragmentActivity(권장), Fragment(권장), FragmentManager, LifeCycle 객체를 요구하므로, 이들 중 하나를 넘겨준다
FragmentStateAdapter를 자세히 살펴보면 RecyclerView.Adapter를 상속받는다. ViewPager2는 RecyclerView와 동일한 원리로 작동한다

  • getItemCount: Adapter가 가지고 있는 data set안에서의 전체 아이템 수를 리턴하는 메소드(RecyclerView.Adapter 클래스의 메소드)
  • createFragment: 특정 포지션에 연결된 새로운 Fragment를 제공하는 기능을 가진 메소드(FragmentStateAdapter 클래스의 메소드)

BottomNavigation은 컨텐츠 전환 및 탐색을 위한 하단 탭바이다

  • ViewPager2와 연동하여 서브 화면들을 전환한다
  • material design 가이드상 BottomNavigation으로 표현하는 화면은 3개 이상 5개 이하(권장)
  • BottomNavigation이 보여주는 화면은 동등한 중요도(깊이)를 가짐

5.BottomNaviagtion과 ViewPager2 연동

private fun initBottomNavi(){
        binding.vpMain.registerOnPageChangeCallback(object: ViewPager2.OnPageChangeCallback(){
            override fun onPageSelected(position: Int) {
                binding.bnvMain.menu.getItem(position).isChecked = true
            }
        })

        binding.bnvMain.setOnItemSelectedListener {
            when(it.itemId){
                R.id.menu_android -> {
                    binding.vpMain.currentItem = FIRST_FRAGMENT
                    return@setOnItemSelectedListener true
                }
                R.id.menu_list -> {
                    binding.vpMain.currentItem = SECOND_FRAGMENT
                    return@setOnItemSelectedListener true
                }
                else -> {
                    binding.vpMain.currentItem = THIRD_FRAGMENT
                    return@setOnItemSelectedListener true
                }
            }
        }

    }

companion object{
    const val FIRST_FRAGMENT = 0
    const val SECOND_FRAGMENT = 1
    const val THIRD_FRAGMENT = 2
}
  • OnPageChangeCallBack
    • ViewPager2의 화면전환을 감시하는 추상 클래스
    • 기존의 ViewPager의 addPageChangeListener가 인터페이스여서 메소드를 모두 재정의 해야했던 반면 ViewPager2의 onPageChangeCallBack은 추상 클래스이기 때문에 필요한 메소드만 재정의하면 됨
  • onItemSelectedListener
    • BottomNavigation의 Item들이 선택되었을 때 호출되는 리스너
    • 현재 코드에서는 어떤 메뉴 아이템이 선택되었는지에 따라 ViewPager2의 CurrentItem(현재 화면에 비춰지는 아이템의 위치)을 바꿔주고 있다
  • companion object은 이벤트 처리 및 가독성을 위한 상수 선언

TabLayout

TabLayout이은 컨텐츠 전환 및 탐색을 위한 탭 레이아웃이다

  • 상단탭을 만들 때 주로 사용
  • BottomNavigationView에 비해 위치 이동이 자유로움
  • CoordinatorLayout, AppBarLayout과 함께 사용하면 알아서 크기와 윛를 조절해 줌
  • 텍스트, 아이콘 뿐 아니라, 커스텀뷰를 메뉴로 설정 가능

TabLayout의 다양한 속성들

  • app: tabGravitiy(탭의 정렬 방식 옵션)
    • fill: 탭의 너비를 동일한 간격으로 한다
    • center: 탭의 가운데로 정렬
  • app: tabMode
    • fixed: 모든 탭이 나오도록 설정(탭의 표시 방식)
    • scrollable: 탭이 화면 밖으로 나갈 경우 스크롤
  • app: tabIconTint(아이콘 색상 옵션)
  • app: tabSelectedTextColor(탭이 선택됐을 떄 글자 색 변경)
  • app: tabIndicatorColor(인디케이터의 색 변경)
  • app: tabIndicatorHeight(인디케이터의 높이 변경)

4.ViewPager2와 TabLayout 연동하기

class TestFragment1 : Fragment() {

    private var _binding: FragmentTest1Binding? = null
    private val binding
        get() = _binding ?: error("Binding 이 초기화 되지 않았습니다.")
    private lateinit var sampleTabViewPagerAdapter: SampleTabViewPagerAdapter

    override fun onCreateView(
        inflater: LayoutInflater, container: ViewGroup?,
        savedInstanceState: Bundle?
    ): View? {
        _binding = DataBindingUtil.inflate(inflater, R.layout.fragment_test1, container, false)

        initAdapter()
        initTabLayout()

        return binding.root
    }

    override fun onDestroyView() {
        super.onDestroyView()
        _binding = null
    }

    private fun initAdapter(){
        val fragmentList = listOf(TabFragment1(), TabFragment2())

        sampleTabViewPagerAdapter = SampleTabViewPagerAdapter(this)
        sampleTabViewPagerAdapter.fragments.addAll(fragmentList)

        binding.vpSample.adapter = sampleTabViewPagerAdapter
    }

    private fun initTabLayout(){
        val tabLabel = listOf("첫 번째", "두 번쨰")

        TabLayoutMediator(binding.tlSample, binding.vpSample) {tab, position ->
            tab.text = tabLabel[position]
        }.attach()

    }

}

TabLayoutMediator는 ViewPager2와 TabLayout을 연동할 떄 사용하는 클래스이다
attach를 호출할 떄 populatreTabsFromPagerAdapter()를 호출하고 여기서 tabLayout.removeAllTabs()로 기존 탭들을 지워버리고 새로운 탭을 생성한다

화면 비율

갤럭시 시리즈 휴대폰만 해도 굉장히 많은데 화면크기, 해상도가 제각각이다
크기가 서로 다른 기기에 디자인이 동일하게 보여지게 하려면 dpi와 dp를 사용한다

dpi(Dot per Inch)

  • 인치당 들어가는 픽셀의 수(화면밀도, 해상도)
  • 10dpi = 1인치에 10개의 도트(픽셀)가 들어간다
  • 72dpi = 1인치에 72개의 도트(픽셀)가 들어간다

px은 화면상의 실제 픽셀에 대응하는 단위이다. 이러한 px단위로 xml상에서 크기 속성르 주면 해상도가 커질수록 이미지가 작아지지만 dp로 크기를 지정하면 전부 똑같은 사이즈로 보인다

주요 밀도 한정자

  • ldpi: 120dpi
  • mdpi: 160dpi(기준 밀도)
  • hdpi: 240dpi
  • xhdpi: 320dpi
  • xxhdpi: 480dpi
  • xxxhdpi: 640dpi

dp(Density Independant Pixel, 밀도 독립형 픽셀)란?

  • 픽셀과 상관 없이 독립적인 크기를 가진다고 불 수 있다
  • 픽셀에 상관 없이 같은 크기를 보여주기 위해 화면 밀도를 알아야 한다
  • 1dp는 중밀도(기준 밀도)화면의 1px과 거의 동일한 가상 픽셀 단위
  • 디스플레이의 해상도(밀도)에 상관없이 다룰 수 있는 단위
  • dp를 사용하면 기기별 해상도가 천차만별인 안드로이드에서 동일한 비율로 크기를 지정할 수 있다
  • 해상도 변환식을 사용하는건 어렵기 때문에 가급적 기준 밀도인 mdpi로 디자인을 달라고 요청하는게 좋다(mdpi의 표준 사이즈는 360 * 640 px 이다)

mdpi 사이즈로 가상머신을 만들고 Invalidate Caches / Restart를 실행하면 LayoutEditor의 프리뷰가 가상머신과 동일해진다

디자이너와 협업 준비

디자이너의 작업물로 개발을 진행하려면 대표적인 디자인 협업 툴인 Zeplin이나 Figma를 사용한다

Zeplin은 디자이너가 작업한 디자인을 바탕으로 개발자와 원할하게 소통할 수 있도록 도와주는 핸드오브 툴이다(피그마보다는 좀 더 개발자 친화적인 툴이다)
요소를 클릭하고 다른 요소에 마우수를 이동시키면 상대적인 거리가 나온다(margin값 사용)
요소를 클릭하면 가로,세로 길이가 나오지만 고정값을 줄 경우 기종에 따라 크기가 달라질 수 있다. 그래서 가급적 해상도에 크게 영향을 받지 않는 속성인 match_parent, wrap_content, 0dp 세가지를 자주 활용하자
제플린에는 해상도별 이미지가 준비되어 있다

피그마는 실시간 협업이 가능한 UX/UI 디자인 툴이다 (개발자를 위한 툴은 아니지만, 한번에 많은 뷰를 볼 수 있다)
Edit권한이 있는 경우 Design 탭까지 보여 디자인 수정이 가능하다. View권한만 있는 경우 inspect탭(정보 탭)까지만 보여 디자인 수정이 불가능하다
Edit 권한이 있는 경우 요소를 클릭하고 Alt키를 누른 채로 다른 요소에 갖다대면 Margin값이 보인다. View 권한만 있는 경우 제플린과 마찬가지로 요소를 클릭하고 다른 요소에 갖다대면 Margin 값이 보인다
피그마에서는 dpi(해상도)별 이미지를 지원하지 않지만 SVG, 9-Patch 두가지 방법으로 해결할 수 있다

PNG로만 Export 가능한 경우 9-Patch를 사용한다. 9-Patch는 뷰의 콘텐츠와 화면 크기를 수용하기 위해 크기가 자동으로 조정되는 비트맵 이미지를 생성할 수 있는 도구이다
NinePatch 이미지를 생성하려는 png이미지를 마우스 오른쪽 버튼 클릭하고 Create 9-Patch File을 클릭 이때 확장자가 .9.png인 이미지 파일을 생성하고 나면 원본 png 파일은 삭제

  • 나인패치는 png 파일에만 적용된다
  • 하나의 이미지로 다양한 비율에 대해 일정한 모양을 유지한다

앱 리소스 알아보기

리소스란 코드를 제외한 앱에서 사용할 외부 파일들(이미지, 애니메이션 등), 앱에서 사용할 자원들이다

  • drawble: 이미지, 도형과 같은 화면에 그릴 수 있는 파일
  • layout: 사용자 인터페이스 레이아웃을 정의하는 XML 파일
  • mipmap: 런처 아이콘 관련 파일
  • values: 문자열, 정수, 색과 같은 단순 값이 들어오는 파일
    • 리소스 배열을 위한 arrays.xml
    • 색상값을 위한 colors.xml
    • 치수값을 위한 dimens.xml
    • 문자열값을 위한 strings.xml
    • 스타일을 위한 styles.xml
    • 앱내테마를 위한 themes.xml

리소스의 경우 xml 내부에서는 @리소스 폴더 이름/res_id로 접근 (android:src=”@drawable/img_github”)
코틀린 코드 내부에서는 R 클래스에서 발생하는 리소스 ID로 액세스 할 수 있다 (R.drawable.img_github)

간단한 도형은 디자이너의 이미지를 다운받아서 사용해도 지만만 Drawable에서 만들어 쓸 수도 있다

ShapeDrawable: 간단한 도형들을 그린 drawable

  • Shape - 도형의 모양
  • Corners - 도형의 굴곡
  • Stroke - 도형의 테두리
  • Solid - 도형의 색깔
  • Gradient - 도형에 그래디언트 적용

예를 들어 모양은 사각형, Radius는 5dp Colors는 보라색이면 아래 코드 처럼 만들면 된다

<?xml version="1.0" encoding="utf-8"?>
<shape xmlns:android="http://schemas.android.com/apk/res/android"
    android:shape="rectangle">
    <solid android:color="#8040FF" />
    <corners android:radius="5dp" />
</shape>

StateListDrawable

입력을 시작하면 모양이 바뀌게 하고 싶으면 객체의 상태에 따라 여러 다른 이미지나 색상을 보여주는 StateListDrawable을 사용하면 된다

  • Selector를 활용해 표현
  • 각 상태에 대한 이미지들은 item 요소로 표현된다
<?xml version="1.0" encoding="utf-8"?>
<selector xmlns:android="http://schemas.android.com/apk/res/android">
    <item android:drawable="@drawable/shpae_edit" android:state_focused="true"/>
    <item android:drawable="@drawable/shape_unselect_edit" android:state_focused="false"/>
</selector>
  • state_enabled: 현재 사용이 가능한지 유무
  • state_selected: 현재 선택되었는지 유무
  • state_focused: 뷰가 포커스 되었는지 유무
  • state_pressed: 현재 눌렀는지 유무
  • state_checked: 현재 체크되어 있는지 유무

텍스트 디자인

android:textSize 텍스트뷰의 글씨 사이즈 지정 단위는 sp(안드로이드 xml test의 default 단위)
android:lineSpacingExtras Line height - size의 크기
android:textColor 텍스트의 색상 입력(hex 코드)

TextView에는 뷰에 폰트를 적용하는 fontFamily라는 속성이 있다

폰트 적용하는 법

  1. 사용할 폰트 다운로드 (Noto_Sans KR)
  2. font 폴더 생성 후 다운받은 폰트 import
  3. fontFamily 파일 만들고 View에 적용

FontWeight

font Family에 있는 폰트들을 fontweight를 이용해 적용

  • font의 글씨 굵기를 설정하기 위해서도 사용
  • fontWeight는 100 ~ 900 사이의 숫자 (100단위로 나누는게 좋음)
<?xml version="1.0" encoding="utf-8"?>
<font-family xmlns:android="http://schemas.android.com/apk/res/android">
    <font
        android:font="@font/notosanskr_black"
        android:fontWeight="400"/>
</font-family>

android:font -> 다운받은 폰트
android:fontWeight -> View에서 폰트를 구별할 값(굵기)
android:fontStyle -> 이태릭, 굵기 등 변경

font family는 안드로이드 8.0 (API 26)부터 사용 가능하다

View 에서는 아래 코드처럼 사용한다

android:fontFamily="@font/noto_sans_kr"
android:textFontWeight="700"

font family 파일을 적용하고 Weight를 넣어주면 Weight 값에 해당하는 폰트가 적용된다

라이브러리(Glide)

  • 프레임워크가 제공하지 않는 기능들을 제공
  • 재사용성을 높이고 반복적인 코드를 줄이기 위해 사용
  • 상황에 맞는 적절한 라이브러리를 사용하면 개발이 무척 편리해짐

안드로리드의 대표적인 이미지 처리 라이브러리로 Glide가 있다

  • 타 이미지 라이브러리에 비해 빠른속도 및 안정성 보장
  • 썸네일 보기, GIF 로딩, 동영상 스틸보기 기능 지원
  • 사용하기 쉽고 간편하다

1.build.gradle(app)에 라이브러리 implementation 하기

implementation 'com.github.bumptech.glide:glide:4.13.0'
annotationProcessor 'com.github.bumptech.glide:compiler:4.13.0'

2.레이아웃에 Glide로 이미지를 띄워줄 imageView 배치

3.인터넷에서 이미지를 불러오기 위해 Mamnifest에 인터넷 권한 추가

<uses-permission android:name="android.permission.INTERNET"/>

4.코드에서 Glide 이용해서 동그란 이미지 만들기

Glide.with(this)
    .load("https://avatars.githubusercontent.com/u/89780201?s-200&v=4")
    .circleCrop()
    .into(binding.ivProfile)
  • with(): View, Fragment 혹은 Activity로 부터 Context를 가져온다
  • load(): Bitmap, Drawable, String, Uri, Url, File, ResourId(Int), ByteArray 등과 같이 다양한 이미지 파일을 로드한다
  • into(): 이미지를 보여줄 View를 저장한다
  • circleCrop(): 보여줄 이미지를 원형으로 만든다
